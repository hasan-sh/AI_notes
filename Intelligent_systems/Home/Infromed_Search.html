Informed Search

Created dinsdag 05 november 2019


Best-first search


node is selected for expansion based on an
evaluation function f(n)

Heuristic functions

The estimated cost of the best path from one node to another.

  • usually abbreviated with h(x)
  • Should have complete knowledge
  • Use partial knowledge/human expertise
  • It could also be (completely) wrong.


Techniques

Best-first search

  • SLD: straight lines systems; finding the straightest line between two nodes(e.g. two cities)
  • Greedy best-first search:
    • Keep all open nodes in the Fringe; In case that is inefficient, restrict the size of the Open List = BeamSearch!
    • Hill climbing; fast, but might not always find a solution
  • A and A*:
    • A*= A search, but with an admissible heuristic
      • A heuristic is admissible if it never overestimates the cost to reach the goal.
    • g(n) the cost (so far) to reach the node; current state, or the steps you have taken so far.
    • h(n) estimated cost to get from the node to the goal.
    • f(n) estimated total cost of path through n to goal.
