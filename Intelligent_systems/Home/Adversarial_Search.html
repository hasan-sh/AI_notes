Adversarial Search

Created woensdag 06 november 2019

Minmax: one player is maximizing their success and the other is minimizing their loss.

Schnapsen phase 2
  • if current node is max; you maximize all possible values after your opponent plays a certain card
  • if current node is min; you minimize all possible values after your opponent plays a certain card



The figure above describes how MAX maximizes the values to start with value of 3. It checks all successors of the MIN(i.e. how many options it has and the minimum value it would choose) then backtrack the values and chooses the maximum one.
Performance is measured in

  • Completeness; does it always find a solution if exists? yes
  • Optimality; does it always find the least-cost solution? n
  • Time complexity; number of nodes checked/expanded?
    • would be expressed in: b^m
  • Space complexity; number of nodes stored in the state representation?
  • would be expressed in: b times m = bm



Some games are so complex which make.s the state space so huge. Cutting off search can help with that:

  • fixed-depth limited search; how many steps you try to predict; that's how many steps your opponent could play

Heuristic Eval

  • should be fast
  • only useful for quiescent
  • Performance of the Heuristic depends on the Heuristic Eval function


Horizon effect: after a huge set of moves, the next state would not be possible to predict; in chess for instance.


WIth partial knowledge stage, a belief states helps with making decisions. E.g. in Schnapsen phase one, we have imperfect knowledge of the other player's and stock's cards. So, we could make assumptions with each card we could play and how the belief state will change after taking that action.
If we reach the goal state, then the action we took is good enough


Alpha-beta pruning


Pruning is the case when a part of the expanded node isn't explored, thus eliminated, because of a condition that had already been explored/examined. This is quit beneficial, because less iterations are made.
